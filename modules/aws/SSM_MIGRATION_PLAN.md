# SSM Migration Plan — Legacy Path Recovery

**Date**: 2026-03-18
**Status**: In progress — resuming tomorrow

---

## Problem

Two sets of secrets were being written to different paths:

1. **Old `modules/secrets/`** — wrote to AWS Secrets Manager under `{name_prefix}-{environment}-langsmith` (no `/langsmith/` prefix). This module has been removed.
2. **`setup-env.sh`** — writes to SSM Parameter Store under `/langsmith/{name_prefix}-{environment}/<key>`

SSM parameters currently exist at `/{name_prefix}-{environment}-langsmith/<key>` (the old path).
ESO is configured to read from `/langsmith/{name_prefix}-{environment}/<key>` (the correct path).
The ESO IRSA role only has permission to read `/langsmith/{name_prefix}-{environment}/*`.

Result: ESO can't sync `langsmith-config` → pods fail with `CreateContainerConfigError`.

### What we tried during the session

1. Changed `setup-env.sh` line 83 to point at the old path — partial fix
2. Added a second ARN to the ESO IAM policy (`main.tf:219`) for the old path — still erroring when session ended

## Recovery Steps

### Before the call — prep

1. **Gather files** from the existing deployment folder:
   - `aws/infra/terraform.tfvars` — actual config
   - `aws/infra/terraform.tfstate` (or S3 backend location)
   - `aws/helm/values/langsmith-values-overrides.yaml` — generated by `init-overrides.sh`
   - `aws/helm/values/langsmith-values.yaml` — base values
   - Any addon values files enabled (agent-deploys, agent-builder, insights, ha, dev)

2. **Verify `name_prefix` and `environment`** from `terraform.tfvars`:
   ```bash
   grep -E '(name_prefix|environment)\s*=' aws/infra/terraform.tfvars
   ```

3. **List actual SSM parameters** to confirm the old path:
   ```bash
   aws ssm get-parameters-by-path --path "/" --recursive \
     --query 'Parameters[].Name' --output table --region <region> \
     | grep -i langsmith
   ```

### Step 1 — Migrate SSM parameters

Run the migration script (dry run first):

```bash
# From the aws/ directory
# Replace <old-prefix> with whatever Step 3 above reveals
./infra/scripts/migrate-ssm.sh --old-prefix /<old-prefix> --dry-run
```

Review the output, then execute:

```bash
./infra/scripts/migrate-ssm.sh --old-prefix /<old-prefix>
```

This copies all params from the old path → `/langsmith/{name_prefix}-{environment}/<key>`.

### Step 2 — Revert the IAM policy hack

In `aws/infra/main.tf` (line ~219), revert back to the single ARN:

```hcl
Resource = "arn:aws:ssm:${var.region}:*:parameter/langsmith/${local.base_name}/*"
```

Remove the temporary second ARN we added for the old path.

Then apply:

```bash
terraform -chdir=infra apply
```

### Step 3 — Revert setup-env.sh

Make sure line 83 is back to the standard path (no `-langsmith` suffix):

```bash
local _path="${_ssm_prefix}/${ssm_name}"
```

### Step 4 — Re-source secrets and validate

```bash
source ./infra/setup-env.sh
./infra/scripts/manage-ssm.sh validate
```

All required params should show `OK`.

### Step 5 — Force ESO re-sync and redeploy

```bash
# Force ESO to pick up the new params immediately
kubectl annotate externalsecret langsmith-config -n langsmith \
  force-sync=$(date +%s) --overwrite

# Verify sync
kubectl get externalsecret langsmith-config -n langsmith

# Redeploy
./helm/scripts/deploy.sh
```

### Step 6 — Clean up old params (optional, after everything works)

```bash
./infra/scripts/migrate-ssm.sh --old-prefix /<old-prefix> --delete-old
```

## If ESO still fails after migration

Check in this order:

1. **Params exist at the right path?**
   ```bash
   ./infra/scripts/manage-ssm.sh list
   ```

2. **ESO can reach SSM?**
   ```bash
   kubectl describe externalsecret langsmith-config -n langsmith
   ```
   Look for `AccessDeniedException` → IAM policy issue
   Look for `Secret does not exist` → wrong SSM path

3. **IRSA trust policy correct?**
   The ESO service account needs `system:serviceaccount:external-secrets:external-secrets`
   ```bash
   kubectl get sa external-secrets -n external-secrets -o yaml
   ```

4. **K8s secret created?**
   ```bash
   kubectl get secret langsmith-config -n langsmith
   ```

5. **Values match?**
   ```bash
   ./infra/scripts/manage-ssm.sh diff
   ```
