Private / Hub-Spoke AKS Landing Zone for LangSmith

What this module builds, what you bring as prerequisites, and the order you deploy it in. Two Terraform roots — infra/ (Azure) and bootstrap/ (in-cluster) — plus a Helm install, bridged through Key Vault with no shared state.

RBAC mode + default-deny · holds connection URLs & app secrets · the "bridge" the other root reads.

03 Deployment sequence

0
Authenticate & pre-flightruns anywhere
az login → scripts/preflight.sh checks login, resource providers, RBAC, and terraform.tfvars.
1
Deploy Azure infrastructureinfra/runs anywhere
terraform apply → private AKS, Postgres, Redis, Blob, Key Vault, bastion, private DNS. No Kubernetes provider, so the private API server never blocks it. ~15–20 min.
2
Cluster access + verifyinfra/runs anywhere
az aks get-credentials (works anywhere); kubectl API calls need VNet connectivity — run them from the jumpbox.
3
In-cluster bootstrapbootstrap/from jumpbox
terraform apply (3 vars) → KEDA, internal NGINX, langsmith namespace, Postgres/Redis connection secrets. Discovers everything via Azure data sources + Key Vault.
3.5
Config & TLS secretsscriptsfrom jumpbox
create-k8s-secrets.sh → langsmith-config-secret (from Key Vault); create-tls-secret.sh → langsmith-tls (self-signed for testing, or BYO cert).
4
Install LangSmith (Helm)Helmfrom jumpbox
helm upgrade --install wiring external Postgres/Redis secrets, Blob via Workload Identity, the config secret, and ingress + TLS. Chart not bundled by this module.
5
Verify & accessHelmfrom jumpbox
status.sh for health; map your config.hostname to the private ingress IP via internal DNS, then reach LangSmith from an in-VNet host.
No shared state. bootstrap/ reads nothing from infra/'s state — it rediscovers via Azure data sources + a Key Vault "secret bridge".
Private Endpoints, not VNet injection. Postgres & Redis use PEs so they stay compatible with user-defined-routing egress.
No cert-manager. langsmith-tls is script-created — self-signed for testing, real cert for production.
ingress-nginx sunset. The bundled open-source controller's upstream maintenance ends March 2026 — plan a successor for long-term use.
